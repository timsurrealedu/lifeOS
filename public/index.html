        When a run hits its usage/rate limit it cascades down this chain: Claude → Kimi → Gemini → DeepSeek. Leave a key empty to skip that link. (Gemini is REST-only, so on write jobs like Process inbox it's skipped and the order there is Kimi → DeepSeek.)
        

      
        Kimi fallback (1st)
        
      
        Kimi (Moonshot) exposes an Anthropic-compatible endpoint, so the same workflow keeps running. Get a key at platform.moonshot.ai.
        
      
      
      
        
      

      
        ChatGPT fallback (read-only, after Kimi)
        
      
        OpenAI / ChatGPT fallback for tutor, chat, add-to-note, and semantic search. REST-only, so write jobs skip it.
        
      
      

      
        Gemini tutor primary
        
      
        A free Google AI Studio key is used first for the per-note tutor, code tutor, and ➕ Add to note. If it runs out, the tutor falls back through Kimi → ChatGPT → Qwen → DeepSeek.
        
      
      

      
        Qwen fallback (after ChatGPT)
        
      
        DashScope Claude Code proxy. Anthropic-compatible, but the proxy has recently returned 500s for this project.
        
      
      
      

      
        DeepSeek / GLM fallback last
        
      
        DeepSeek and GLM also expose Anthropic-compatible endpoints, so they cover write jobs too. Leave the key empty to disable.
        
      
      
      

      
        Default AI provider

Tutors still try Gemini first. Stewie sync supports Claude, Kimi, and DeepSeek with its current provider code.
